# 飞书多维表格配置检查清单

## ✅ 快速开始（5分钟完成）

### 1. 创建多维表格（2分钟）
- [ ] 登录飞书 → 点击「多维表格」
- [ ] 新建空白表格，命名为「网站询盘管理」
- [ ] 添加以下字段：
  - [ ] 提交时间（日期时间）
  - [ ] 姓名（文本）
  - [ ] 联系电话（文本）
  - [ ] 公司（文本）
  - [ ] 咨询类型（单选）
  - [ ] 留言内容（多行文本）
  - [ ] 处理状态（单选，默认：待跟进）

### 2. 配置 Webhook 自动化（2分钟）
- [ ] 点击右上角「自动化」
- [ ] 创建自定义流程
- [ ] 触发条件选择「接收到 webhook 时」
- [ ] **复制 Webhook URL**（重要！）
- [ ] 执行动作选择「新增记录」
- [ ] 字段映射：
  - [ ] 姓名 ← `{{webhook.body.姓名}}`
  - [ ] 联系电话 ← `{{webhook.body.联系电话}}`
  - [ ] 公司 ← `{{webhook.body.公司}}`
  - [ ] 咨询类型 ← `{{webhook.body.咨询类型}}`
  - [ ] 留言内容 ← `{{webhook.body.留言内容}}`
  - [ ] 提交时间 ← 「当前时间」
  - [ ] 处理状态 ← 默认值「待跟进」
- [ ] 保存并开启自动化

### 3. 更新网站代码（1分钟）
- [ ] 打开 `contact.html`
- [ ] 找到 `FEISHU_CONFIG` 配置
- [ ] 将 `WEBHOOK_URL` 替换为复制的地址：
```javascript
const FEISHU_CONFIG = {
  WEBHOOK_URL: 'https://open.feishu.cn/open-apis/bitable/v1/webhooks/xxxxxxxxxx',
  TOKEN: ''
};
```

### 4. 测试验证
- [ ] 打开网站联系页面
- [ ] 填写测试数据提交
- [ ] 检查飞书表格是否收到数据
- [ ] ✅ 完成！

## 🐛 常见问题

### 问题1：提交后没有反应
**检查项：**
- [ ] Webhook URL 是否正确复制（不要有多余空格）
- [ ] 自动化流程是否已开启
- [ ] 浏览器控制台是否有报错（F12 → Console）

### 问题2：数据没有写入表格
**检查项：**
- [ ] 字段名是否完全匹配（区分大小写）
- [ ] 字段映射配置是否正确
- [ ] 网络请求是否成功（F12 → Network）

### 问题3：部分字段为空
**检查项：**
- [ ] 字段名是否与代码中的 `submitData` 一致
- [ ] 飞书表格字段名是否有特殊字符或空格

### 问题4：跨域错误（CORS）
**说明：** 飞书 Webhook 支持跨域，一般不会出现此问题
**解决：** 确保使用 HTTPS 访问网站

## 📊 数据管理建议

### 每日查看
- 筛选「处理状态 = 待跟进」查看新询盘
- 及时分配给负责人

### 每周统计
- 使用「数据透视表」统计各咨询类型数量
- 分析转化率

### 每月备份
- 导出数据到 Excel 备份
- 路径：多维表格 → ··· → 导出为 Excel

---

## 🔒 安全建议

1. **不要**将 Webhook URL 暴露在公开代码仓库
2. **建议**开启飞书的「凭证校验」功能
3. **定期**检查表格访问权限
4. **敏感数据**（如手机号）在分享时脱敏处理
