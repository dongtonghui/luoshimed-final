# Google Apps Script 授权步骤详解

## 什么时候需要授权

首次部署或重新部署时，Google会要求授权，因为脚本需要：
- 访问你的 Google Sheets
- 发送邮件（如果配置了邮件通知）

---

## 详细授权步骤

### 步骤1: 点击部署

1. 在 Google Apps Script 编辑器中
2. 点击右上角的 **部署** 按钮
3. 选择 **新建部署**

```
┌─────────────────────────────────────┐
│  部署 ▼                              │
├─────────────────────────────────────┤
│  测试部署                            │
│  ├─ 开发环境                         │
│  └─  API 可执行文件                   │
│  ─────────────────                  │
│  新建部署                            │  ← 点击这个
│  管理部署                            │
└─────────────────────────────────────┘
```

---

### 步骤2: 配置部署选项

在弹出的对话框中：

```
┌─────────────────────────────────────┐
│  新建部署                            │
├─────────────────────────────────────┤
│  类型: [Web应用 ▼]                   │  ← 必须选这个
│                                     │
│  描述: 网站询盘表单接口               │
│                                     │
│  执行身份: [我 ▼]                    │  ← 选择"我"
│                                     │
│  访问权限: [任何人 ▼]                │  ← 必须选"任何人"
│                                     │
│           [  部 署  ]                │  ← 点击部署
└─────────────────────────────────────┘
```

**关键设置：**
- ✅ 类型：**Web应用**
- ✅ 执行身份：**我**
- ✅ 访问权限：**任何人**

---

### 步骤3: 授权访问（关键步骤）

点击部署后，会弹出授权对话框：

```
┌─────────────────────────────────────┐
│  需要授权                            │
├─────────────────────────────────────┤
│  此项目需要访问您的 Google 账号      │
│  才能运行。                          │
│                                     │
│     [  取 消  ]   [ 授权访问 ]       │  ← 点击"授权访问"
└─────────────────────────────────────┘
```

**点击 [授权访问]**

```

**不要点击返回！** 而是：

1. 点击左下角的 **"高级"**

```
┌─────────────────────────────────────┐
│  Google 尚未验证此应用               │
├─────────────────────────────────────┤

│                                     │

│                                     │
│  高级 ▼                             │  ← 点击"高级"
└─────────────────────────────────────┘
```

2. 点击后展开：

```
┌─────────────────────────────────────┐
│  Google 尚未验证此应用               │
├─────────────────────────────────────┤
│                                     │
│  此应用尚未经过 Google 验证...       │
│                                     │
│  [  返 回  ]                        │
│                                     │

---

### 步骤7: 复制Web应用URL

授权完成后，会显示：

```
┌─────────────────────────────────────┐
│  部署已完成                          │
├─────────────────────────────────────┤
│                                     │
│  Web应用                            │
│  ─────────────────────────────────  │

│                                     │
│        [  完 成  ]                  │
└─────────────────────────────────────┘
```

**点击复制URL**，然后点击 **完成**

## 验证授权是否成功

### 方法1: 浏览器直接测试

复制URL到浏览器访问，应该显示：
```json
{"status":"ok","message":"服务运行正常"}
```

### 方法2: 查看执行日志

1. 在Apps Script编辑器左侧
2. 点击 **执行** 图标（▶️）
3. 查看是否有执行记录

## 常见问题

### Q: 没有看到授权弹窗？

可能原因：
- 之前已经授权过
- 部署类型不是"Web应用"
- 使用的是同一个账号

### Q: 授权后还是无法访问？

检查：
1. 是否点击了"允许"（不只是"授权访问"）
2. 是否完成了"高级"→"前往..."→"允许"全部步骤
3. URL是否以 `/exec` 结尾

### Q: 如何重新授权？

1. 访问 https://myaccount.google.com/permissions
2. 找到你的脚本项目
3. 点击 **移除访问权限**
4. 重新部署并授权

### Q: 授权时显示"此应用已被阻止"？

这是Google的安全策略，尝试：
1. 使用个人Google账号（不是企业/教育账号）
2. 或者使用替代方案（Formspree等）

---

## 如果授权后仍然失败

请提供以下信息：
1. 部署类型是什么？
2. 是否完成了"高级"→"前往..."→"允许"步骤？
3. 浏览器直接访问URL显示什么？
4. 执行日志中有什么错误？
