# 案例页重构实现计划

**方案：** 多页面结构（方案A）  
**日期：** 2025-03-20  
**目标：** 将案例页重构为支持子菜单导航、JSON数据驱动、博客样式详情的结构

---

## 文件结构变更

```
luoshi-website/
├── cases.html                    # 案例总览页（改造）
├── cases-medical.html            # 医用系列案例集合页（新增）
├── cases-home.html               # 家用系列案例集合页（新增）
├── case-detail.html              # 案例详情页（新增）
├── data/
│   └── cases.json               # 案例数据配置（新增）
└── css/
    └── nav-dropdown.css         # 导航下拉菜单样式（新增）
```

## 任务列表

### Task 1: 创建案例数据 JSON 文件
**时间：** 5分钟  
**文件：** `data/cases.json`

内容：
- 医用系列案例数组（5-6个示例案例）
- 家用系列案例数组（4-5个示例案例）
- 每个案例包含：id, series, title, subtitle, cover, summary, metrics, content, tags

验收标准：
- [ ] JSON 格式合法
- [ ] 包含医用和家用两个系列
- [ ] 每个案例有完整的字段

---

### Task 2: 创建导航下拉菜单样式
**时间：** 5分钟  
**文件：** `css/nav-dropdown.css`（或在共享样式中添加）

内容：
- 下拉菜单容器样式
- 悬停触发下拉效果
- 子菜单项样式
- 移动端适配

验收标准：
- [ ] 悬停"合作案例"显示下拉菜单
- [ ] 下拉菜单包含"医用系列"和"家用系列"
- [ ] 样式与现有导航一致
- [ ] 移动端正常显示

---

### Task 3: 更新所有页面的导航栏
**时间：** 10分钟  
**修改文件：** 所有 HTML 文件的导航部分

修改内容：
- 将 `<a href="cases.html">合作案例</a>` 改为下拉菜单结构
- 悬停显示子菜单：医用系列、家用系列
- 保持现有 active 状态逻辑

验收标准：
- [ ] 所有页面的导航栏一致更新
- [ ] 悬停显示下拉菜单
- [ ] 子菜单链接正确

---

### Task 4: 创建医用系列案例集合页
**时间：** 10分钟  
**文件：** `cases-medical.html`

内容：
- 复用现有案例页的 banner 和样式
- 页面标题：医用系列案例
- 使用 JavaScript 从 cases.json 加载医用系列案例
- 使用现有 `.case-card` 样式渲染卡片
- 点击卡片跳转到 `case-detail.html?id=xxx`

验收标准：
- [ ] 页面加载时显示所有医用案例
- [ ] 卡片样式与现有案例页一致
- [ ] 点击卡片正确跳转详情页
- [ ] 面包屑导航正确

---

### Task 5: 创建家用系列案例集合页
**时间：** 10分钟  
**文件：** `cases-home.html`

内容：
- 与医用系列页结构相同
- 加载家用系列案例数据
- 页面标题：家用系列案例

验收标准：
- [ ] 页面加载时显示所有家用案例
- [ ] 卡片样式一致
- [ ] 点击卡片正确跳转

---

### Task 6: 创建案例详情页（博客样式）
**时间：** 15分钟  
**文件：** `case-detail.html`

内容：
- 博客样式布局：
  - 顶部：大封面图 + 标题 + 发布信息
  - 正文：富文本内容区（支持段落、小标题、引用、图片）
  - 侧边栏：相关案例推荐
  - 底部：返回按钮
- 从 URL 参数获取 case id
- 从 cases.json 加载对应案例数据
- 响应式布局

验收标准：
- [ ] 根据 URL 参数正确显示案例
- [ ] 博客样式美观易读
- [ ] 封面图、标题、正文正确渲染
- [ ] 移动端适配良好

---

### Task 7: 改造案例总览页

验收标准：
